Estándar de Jerarquía del Sistema de Archivos Linux

El Filesystem Hierarchy Standard (FHS; Estándar de Jerarquía del Sistema de Archivos) es un estándar formal. Es mantenido por la Linux Foundation y define la estructura y los contenidos principales de los directorios en sistemas operativos Linux y similares a UNIX. Su propósito es proporcionar una estructura de sistema de archivos coherente para facilitar la interoperabilidad y la compatibilidad entre sistemas y aplicaciones.

Los siguientes directorios, o link simbólicos a directorios, son necesarios en el / del sistema.

Directorio Descripción
bin Essential command binaries
boot Static files of the boot loader
dev Device files
etc Host-specific system configuration
home User home directories (optional)
lib Essential shared libraries and kernel modules
lib<qual> Alternate format essential shared libraries (optional)
media Mount point for removable media
mnt Mount point for mounting a filesystem temporarily
opt Add-on application software packages
root Home directory for the root user (optional)
run Data relevant to running processes
sbin Essential system binaries
srv Data for services provided by this system
tmp Temporary files
usr Secondary hierarchy
var Variable data
