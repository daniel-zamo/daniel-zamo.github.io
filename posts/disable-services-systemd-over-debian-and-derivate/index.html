Deshabilitar servicios basados en systemd

En este art√≠culo se muestran como detener algunos servicios que no se utilizaran sobre un sistema basado en Ubuntu, donde systemd es el gestor e inicializalizador de los servicios.

Nota: en este ejemplo se utiliza un sistema Ubuntu 22.04 pero los siguientes comandos son validos para Debian o para cualquier otro Linux que este utilizando a systemd propiamente dicho.

# Desabilito servicios   
sudo systemctl disable --now apparmor.service cloud-config.service \
  cloud-final.service cloud-init-local.service cloud-init.service \
  ModemManager.service packagekit.service snapd.apparmor.service \
  snapd.seeded.service snapd.service snapd.socket ufw

Comando systemctl …. service

sudo systemctl enable <nombre_servicio>.service
sudo systemctl enable <nombre_servicio>

sudo systemctl start <nombre_servicio>

sudo systemctl stop <nombre_servicio>

sudo systemctl reload <nombre_servicio>

systemctl status <nombre_servicio>

systemctl is-enabled <nombre_servicio>

systemctl is-active <nombre_servicio>
