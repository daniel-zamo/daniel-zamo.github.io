Mi Nextcloud interno con Docker (Parte I)

Posted Aug 22, 2023
By Daniel Oscar Zamo
2 min read

En esta entrada se despliegan 2 aplicaciones de docker. De este modo se muestra como desplegar diferentes ambientes de la misma, para implementar, por ejemplo, multiples entornos (producción, otra para desarrollo, etc).

Para la creación y ejecución de las aplicaciones contenerizadas, se utilizá docker compose.

Además las variables de entorno para la contrucción de las imágenes han sido separadas de los propios ficheros docker-compose.yml.

Nota del autor: Los ficheros de variables de entorno (o environment), en este caso de nombre .dev.env y .env, no es compartido en este repositorio. Si se muestra la estructura en este documento. (Dichos archivos son básicamente ficheros de definición de variables del estilo interpretado por el shell bash).

Repositorio de esta aplicación

En este URL se comparte los ficheros de esta aplicación implementada.

Consideraciones cumplidas

  • Para este despliegue se da por supuesto que se tiene un host anfitrión motor de Docker (o Podman) disponible.
  • Se utilizarán volumenes anónimos.
  • Cada una de las funcionalidades básicas necesarias (http y base de datos) son implementadas sobre contenedores independientes.

Tecnologías usadas

  • http: apache.
  • Nextcloud (PHP + MariaDB).

# Fichero 'docker-compose.dev.yml'

version '3'
volumes:
  dev_nextcloud:
  dev_db:

services:
  dev_db:
    image: mariadb:10.6
    restart: always
    command: --transaction-isolation=READ-COMMITTED --log-bin=binlog --binlog-format=ROW
    volumes:
      # En este caso uso volumenes anónimo, que
      ### se encontrará en este mismo directorio
      - ./dev_db:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - MYSQL_DATABASE=${MYSQL_DATABASE}
      - MYSQL_USER=${MYSQL_USER}

  dev_app:
    image: nextcloud
    restart: always
    ports:
      - ${PUBLISH_APP}:80
    links:
      - dev_db
    volumes:
      # En este caso uso volumenes anónimo, que
      ### se encontrará en este mismo directorio
     - ./dev_nextcloud:/var/www/html
    environment:
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - MYSQL_DATABASE=${MYSQL_DATABASE}
      - MYSQL_USER=${MYSQL_USER}
      - MYSQL_HOST=${MYSQL_HOST}

volumes:
  nextcloud:
  db:

services:
  db:
    image: mariadb:10.6
    restart: always
    command: --transaction-isolation=READ-COMMITTED --log-bin=binlog --binlog-format=ROW
    volumes:
      # En este caso uso volumenes anónimo, que
      ### se encontrará en este mismo directorio
      - ./db:/var/lib/mysql
    environment:
      - MYSQL_ROOT_PASSWORD=${MYSQL_ROOT_PASSWORD}
      - MYSQL_PASSWORD=${MYSQL_PASSWORD}
      - MYSQL_DATABASE=${MYSQL_DATABASE}
      - MYSQL_USER=${MYSQL_USER}

  app:
    image: nextcloud
    restart: always
    ports:
      - ${PUBLISH_APP}:80
    links:
      - db
    volumes:
      # En este caso uso volumenes anónimo, que
      ### se encontrará en este mismo directorio
     - ./nextcloud:/var/www/html
    environment:
