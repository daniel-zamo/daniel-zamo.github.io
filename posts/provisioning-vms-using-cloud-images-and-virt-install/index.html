Aprovisionamiento de VMs basadas en la nube (cloud-init), con virt-install

Posted Aug 31, 2023
By Daniel Oscar Zamo
3 min read

En esta publicación se muestra como desplegar una VMs (instancia de Ubuntu 22.04 LTS), utilizando Cloud Image sobre un host anfitrión de virtualización Libvirt/KVM, utilizando el CLI virt-install.

En el repositorio https://gitlab.com/dzamo/provisioning-cloud-base-vms-using-cli-and-virt-install se comparten algunos aprovisionamientos de VM Cloud images, realizadas sobre un host anfitrión de virtualización Libvirt/KVM, mediante el CLI de virt-install

En particular el caso de uso …04.ubuntu-jammy es el despliegue de una VM basada en la distribución Ubuntu 22.04 LTS.

Especificación del script implementado

Se deja la implementación del script principal, que realiza la propia tarea de definición de los ficheros:

  • ./${vm_name}-meta-data
  • ./${vm_name}-user-data

url_images='https://cloud-images.ubuntu.com/jammy/current'
img_name='jammy-server-cloudimg-amd64.img'
my_cloud_base_images_path='../cloud-base-images'

pre_fix="ubuntu-jammy"

vm_name="${pre_fix}-${num_id}"
lan_domain="my.net"

# Inicialmente se utiliza el mismo password, tanto para root como para ${user1}
#pwd_initial='bastioninseguro'

# Se utiliza las mismas claves tanto para el root como para el usuario ${user1}

sudo cp -vf ./${vm_name}-{meta,user}-data /var/lib/libvirt/boot/cloud-init/

sudo cp -fv ${my_cloud_base_images_path}/${img_name} ${libvirt_storage}/${vm_name}.qcow2

    --memory 3072  --cpu host --vcpus 3 \
    --os-type linux --os-variant ${os_variant} --import \
    --disk ${libvirt_storage}/${vm_name}.qcow2,format=qcow2,bus=virtio,size=20  \
    --network network=default,model=virtio \
    --cloud-init user-data="/var/lib/libvirt/boot/cloud-init/${vm_name}-user-data",meta-data="/var/lib/libvirt/boot/cloud-init/${vm_name}-meta-data"
