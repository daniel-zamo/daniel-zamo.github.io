Archetypes en Hugo

  

  • SSG
  • Hugo
  • Markdown
  • Generadores de Sitios Estáticos

Hugo incluye de forma nativa la funcionalidad de poder predefinir el contenido inicial (incluido el front matter) de las entradas que se crean con el comando hugo new este tipo de elementos son los templates Archetypes del generador de contenido estático Hugo.

Configuración de una plantilla Archetypes

Crear un archivo de arquetipo personalizado

En el proyecto Hugo en que se este trabajando, este tipo de plantillas se ubican en la carpeta ROOT_SITE/archetypes. En este ejemplo, y como este sitio que se esta desarrollando utiliza el Theme Toha es que la mayoría del contenido son o se ubican en ROOT_SITE/content/posts es que se crea la plantilla ROOT_SITE/archetype/posts.md.

Definir el front matter en el archivo arquetipo

Dentro del archivo posts.md, se agrega la definición del front matter que se pretenda usar como plantilla. En este ejemplo el front matter se define como:

---
date: '{{ .Date }}'
draft: true
title: '{{ replace .File.ContentBaseName `-` ` ` | title }}'
description: 
# Variables mias
showMyAuthor: false
showMyTime: false
menu:
  sidebar:
    name: 
    identifier: 
    parent: 
    weight: 
hero: images/...
tags: []
#categories:
#- Basic
---

- Entrada creada desde plantilla Archeotype creada para el theme Toha

Donde:

  • {{ .Name }}: Es el nombre usado al crear la entrada (contenido o post, en este caso).
  • {{ .Date }}: Es la fecha actual.
  • Se puedes agregar campos personalizados según las necesidades, como por ejemplo show_author o hide_date.
