Docker-Registry sobre Debian 12 (sin autenticación)

Posted Aug 22, 2023
By Daniel Oscar Zamo
1 min read

En esta entrada se instala Docker-Registry, creando así un host/servidor que se puede utilizar como registro privado de imágenes Docker.

Consideraciones cumplidas

  • Las tareas realizadas para este caso de uso, fueron ejecutadas sobre un servidor host anfitrión con runtime Docker, sobre un sistema que posee distribución Linux Debian 12, ya instalado y disponible.

Tecnologías usadas

  • Distribución Linux del sistema: Debian 12.
  • Host anfitrión con container runtime Docker.

Especificaciones iniciales

  • Nombre de host:
    • debian12-docker
    • mydocker

Instalación de docker-registry

1

sudo apt update && sudo apt -y upgrade && sudo apt -y install docker-registry

Configurar acceso http sin autenticación

La siguiente configuración usa conexión HTTP y sin autenticación.

En el fichero /etc/docker/registry/config.yml se comenta las siguientes líneas para deshabilitar auth. Su contenido queda como a continuación:

# Fichero /etc/docker/registry/config.yml
dzamo@debian12-docker:~$ cat /etc/docker/registry/config.yml
version: 0.1
log:
  fields:
    service: registry
storage:
  cache:
    blobdescriptor: inmemory
  filesystem:
    rootdirectory: /var/lib/docker-registry
  delete:
    enabled: true
http:
  addr: :5000
  headers:
    X-Content-Type-Options: [nosniff]
#auth:
#  htpasswd:
#    realm: basic-realm
#    path: /etc/docker/registry
health:
  storagedriver:
    enabled: true
    interval: 10s
    threshold: 3

dzamo@debian12-docker:~$ cat /etc/docker/daemon.json 
{
  "insecure-registries":
    [
      "docker.internal:5000",
      "mydocker:5000"
    ]
}

dzamo@debian12-docker:~$ sudo systemctl restart docker

Push imágen sobre el localhost

1
2
3

dzamo@debian12-docker:~$ sudo docker pull debian
dzamo@debian12-docker:~$ sudo docker tag debian mydocker:5000/debian:my-registry
dzamo@debian12-docker:~$ sudo docker images
