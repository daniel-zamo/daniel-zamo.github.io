The Filesystem Hierarchy Standard / Estándar FHS de Linux

El Filesystem Hierarchy Standard (FHS; Estándar de Jerarquía del Sistema de Archivos) es un estándar formal. Es mantenido por la Linux Foundation, este define la estructura y los contenidos principales de los directorios en sistemas operativos Linux y similares a UNIX. Su propósito es proporcionar una estructura de sistema de archivos coherente para facilitar la interoperabilidad y la compatibilidad entre sistemas y aplicaciones.

El estándar de jerarquía de sistemas de archivos

El estándar de jerarquía de sistemas de archivos (FHS), administrado originalmente por el Free Standards Group y ahora por The Linux Foundation, especifica los directorios principales que deben estar presentes y describe sus propósitos. El documento del estándar de jerarquía de sistemas de archivos se puede obtener en línea.

Al especificar un diseño estándar, el FHS simplifica las predicciones de las ubicaciones de los archivos. Si bien la mayoría de las distribuciones de Linux respetan el FHS, probablemente ninguna de ellas lo siga exactamente, y la última versión oficial podría no tener en cuenta algunos nuevos desarrollos.

A las distribuciones les gusta experimentar y, con el tiempo, algunos de los experimentos se aceptan de forma general.

Los siguientes directorios, o link simbólicos a directorios, son necesarios en el / del sistema.

Directorio Descripción
/ Primary directory of the entire filesystem hierarchy
/bin Essential command binaries. Executable programs that must be available in single user mode
/boot Static files of the boot loader. Files needed to boot the system, such as the kernel, initrd or initramfs images, and boot configuration files and bootloader programs
/dev Device files. Device nodesm used to interact with hardware and software devices.
/etc Host-specific system configuration. System-wide configuration files.
/home User home directories (optional). User home directories, including personal settings, files, etc.
/lib Essential shared libraries and kernel modules.
/lib Alternate format essential shared libraries (optional). Libraries required by executable binaries in /bin and /sbin.
/lib64 64-bit libraries required by executable binaries in /bin and /sbin, for systems which can run both 32-bit and 64-bit programs.
/media Mount point for removable media. Mount points for removable media such as CDs, DVDs, USB sticks, etc.
/mnt Mount point for mounting a filesystem temporarily.
/proc Virtual pseudo-filesystem giving information about the system and processes running on it. Can be used to alter system parameters.
/opt Add-on application software packages.
/root Home directory for the root user (optional)
/run Data relevant to running processes. Run-time variable data, containing information describing the system since it was booted. Replaces the older /var/run.
/sbin Essential system binaries
/srv Data for services provided by this system. Seldom used.
/sys Virtual pseudo-filesystem giving information about the system and processes running on it. Can be used to alter system parameters. Similar to a device tree and is part of the Unified Device Model.
/tmp Temporary files; on many distributions lost across a reboot and may be a ramdisk in memory.
/usr Multi-user applications, utilities and data; theoretically read-only.
/var Variable data that changes during system operation.

Es posible que se encuentren directorios adicionales específicos de la distribución bajo el directorio raíz (/). Estos pueden incluir /misc, que se puede utilizar para datos diversos, y /tftpboot, que se utiliza para arrancar mediante tftp. Si hay archivos en el directorio, están relacionados con el arranque de estaciones de trabajo sin disco. Tenga en cuenta que tener otros directorios no viola el FHS; sin embargo, no lo cumple el tener componentes en directorios distintos de los dictados por el estándar.
