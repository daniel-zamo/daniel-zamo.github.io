USB Ventoy uso de VentoyPlugson y fix para SystemRescue

Posted Aug 24, 2023 Updated Aug 26, 2023
By Daniel Oscar Zamo
1 min read

En esta entrada se muestra el uso de VentoyPlugson para modificar parámetros, y poder hacer booteable la suite SystemRescue sobre un USB con Ventoy.

Dependencias cumplidas

  • Las tareas realizadas aquí han sido ejecutadas sobre un Desktop Linux basado en Fedora 38, con KDE Plasma.
  • También se da por hecho de que se posee un USB con Ventoy boot ya operativo. Si no es así puede consultar este otro artículo, o consultar el sitio oficial del desarrollador

Tecnologías usadas

  • Distribución Linux del sistema: Fedora 38, con KDE Plasma
  • Ventoy, solución para USB booteable y multiple imágenes.
    • VentoyPlugson.
  • SystemRescue

Ejecutando Ventoy-plugson

La siguiente captura muestra como ejecutar Ventoy-plugson e ingresar a la interface web en la opción Global Control Plugin, para modificar VTOY_LINUX_REMONT de 0 pasarlo a 1.

Modificando configuración

Copiar ISOs

Se copian varias versiones de SystemRescue en formato ISO.

Arranque desde nuestro USB Ventoy

Ya preparado nuestro USB Ventoy, con los ISO necesarios, se bootea desde dicho dispositivo y se selecciona que versión arrancar de SystemRescue.

SystemRescue arranca y presenta su menú. En la captura siguiente se selecciona la Boot SystemRescue using default options.

SystemRescue arranca y queda en el CLI del sistema, mostrando ademas algunas de las opciones de utilización.

Para arrancar el ambiente gráfico se ejecuta startx, y el WM XFCE se ejecutará.
