Deshabilitar servicios basados en systemd

En este artículo se muestran como detener algunos servicios que no se utilizaran sobre un sistema basado en Ubuntu, donde systemd es el gestor e inicializalizador de los servicios. Nota: en este ejemplo se utiliza un sistema Ubuntu 22.04 pero los siguientes comandos son validos para Debian o para cualquier otro Linux que este utilizando a systemd propiamente dicho. # Desabilito servicios sudo systemctl disable --now apparmor.service cloud-config.service \ cloud-final.service cloud-init-local.
