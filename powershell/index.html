Windows PowerShell

d@Zamo enero 01, 2025 Actualizado: febrero 02, 2025 #windows #powershell #sistemas-operativos

PowerShell es un intérprete de línea de comandos y un lenguaje de scripting avanzado desarrollado por Microsoft. A diferencia del clásico Símbolo del sistema (cmd.exe), que está basado en comandos heredados de MS-DOS, PowerShell está construido sobre .NET y trabaja con objetos en lugar de solo texto, lo que lo hace mucho más potente para la administración de sistemas.

Características

¿Por qué es importante en Windows 11?

En Windows 11, PowerShell sigue siendo una herramienta clave para la administración y automatización del sistema.

Puede usarse para:

✅ Configurar y administrar Windows con comandos avanzados.

✅ Ejecutar scripts para tareas repetitivas o automatizar procesos.

✅ Gestionar sistemas locales y remotos.

✅ Manipular el sistema de archivos, servicios, procesos y el registro de Windows.

Desde Windows 10 y 11, Microsoft ha impulsado PowerShell Core (la versión moderna, multiplataforma y basada en .NET Core) con el nombre de PowerShell 7, que es la evolución de Windows PowerShell 5.1.

PowerShell en Windows 11 es la herramienta de automatización y administración avanzada, reemplazo moderno del clásico "cmd", con soporte para scripts complejos, administración remota y ejecución multiplataforma.

Sintaxis de PowerShell

PowerShell usa una sintaxis basada en verbos y nombres que sigue un patrón estándar llamado "Verbo-Nombre", lo que hace que los comandos sean más fáciles de entender y predecir.

Ejemplos:


Get-Process    # → Obtiene los procesos en ejecución.
Stop-Service   # → Detiene un servicio en Windows.
New-Item       # → Crea un nuevo archivo o carpeta.

Estos comandos son llamados cmdlets (se pronuncia command-lets), y están diseñados para trabajar con objetos, en lugar de solo texto como en cmd.exe o Bash.

PowerShell y .NET: Ventaja sobre Windows

PowerShell está desarrollado sobre .NET, lo que le da acceso directo a las API y bibliotecas de .NET. Gracias a esto, PowerShell no solo ejecuta comandos como cmd, sino que puede interactuar con objetos de Windows de manera nativa.

Ejemplo práctico:

$proceso = Get-Process explorer
$proceso.Kill()

Aquí, Get-Process explorer obtiene el proceso del Explorador de Windows como un objeto, y luego usamos .Kill() para cerrarlo, algo que en cmd solo se podría hacer con taskkill /IM explorer.exe.

Al ser orientado a objetos, PowerShell permite:

✅ Manipular configuraciones de Windows fácilmente.

✅ Administrar el registro (Registry), servicios y procesos como si fueran objetos de .NET.

✅ Ejecutar scripts avanzados con estructuras de control (if, foreach, try-catch).

✅ Integrarse con otras tecnologías de Microsoft como Active Directory, Exchange, Azure, etc.

Básicamente, PowerShell no solo ejecuta comandos, sino que trabaja con estructuras y elementos del sistema operativo de forma más avanzada y flexible.
