03 - Fedora 38 - configuración inicial

Configuración mínima inicial, tras instalar Fedora 38

09 July 2023

En este documento se comparte las tareas de configuración inicial/básica (recomendada) para un Linux basado en Fedora 38. Se da por supuesto cumplido la instalación documentada en este otro artículo de este mismo sitio.

Detalle de los comandos

Configuración usuario para sudo

El siguiente comando, configura para el usuario $USER poder ejecutar los comandos con privilegios de usuario root sin que se le pida password.

echo "$USER ALL=(ALL) NOPASSWD:ALL"|sudo tee /etc/sudoers.d/$USER

Configurar usuario para usuario su (admin en wheel)

Con lo siguiente (y para sistemas basados en RHEL) los usuarios pertenecientes al grupo wheel pueda hacer uso del comando su. Esta configuración es considerada como tarea de hardening.

sudo usermod -G wheel -a $USER
sudo sed -i \
  's/^#auth\t\trequired\tpam_wheel.so use_uid$/&\nauth\t\trequired\tpam_wheel.so use_uid/' \
  /etc/pam.d/su

Deshabilitar firewalld y SELinux

sudo sed -i.org 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config

Deshabilitar SELinux

sudo sed -i.org 's/SELINUX=enforcing/SELINUX=disabled/' /etc/selinux/config
# Para que los cambios se ejecuten, es neccesario reiniciar
# sudo reboot

Configuración Dnf (flags)

echo 'fastestmirror=1' | sudo tee -a /etc/dnf/dnf.conf
echo 'max_parallel_downloads=6' | sudo tee -a /etc/dnf/dnf.conf
echo 'deltarpm=true' | sudo tee -a /etc/dnf/dnf.conf

Actualizar paquetes del sistema

sudo dnf upgrade -y --refresh
sudo dnf -y update

Instalación de paquetes y/o servicios (para CLI)

sudo dnf -y install \
    git htop vim \
    rsync openssh-server openssh-clients

Deshabilitar swap sobre zram

sudo systemctl disable --now systemd-zram-setup@zram0.service
sudo dnf remove -y zram-generator zram-generator-defaults
sudo swapoff -a && sudo swapon -a
