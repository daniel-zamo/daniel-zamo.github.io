# Despliegue del Orquestador en Fedora 43

Este procedimiento describe la implementación práctica del [Estándar de Arquitectura de Nodo de Control](./std-ans-controller-arquitecture.mdx) sobre un sistema operativo Fedora 43.

## 1. Instalación Base

Actualizamos el sistema e instalamos el motor de orquestación:

```bash
sudo dnf update -y
sudo dnf install -y ansible-core openssh-clients
```

## 2. Configuración de Salida (Callbacks)

Para mantener la legibilidad, configuramos el formato de salida nativo en YAML dentro del archivo `ansible.cfg`:

```ini
[defaults]
stdout_callback = default
result_format = yaml
interpreter_python = auto_silent
```

## 3. Validación de Conectividad

Antes de operar, verifique que la identidad SSH esté configurada según el estándar definido en la [sección de seguridad de la arquitectura](./std-ans-controller-arquitecture.mdx#3-requisitos-de-infraestructura).

```bash
ansible all -m ping
```
