## 1. Aplicación del Método del Salto Mágico

<Badge text="Técnica Principal" variant="tip" size="small" />

### Ejemplo 1: Red /26

<Tabs>
  <TabItem label="Problema">
    **Red:** 192.168.1.0/26
    **Máscara:** 255.255.255.192
    **Calcular:** Todas las subredes y sus rangos
  </TabItem>
  
  <TabItem label="Cálculos">
    ```bash title="Aplicando el salto mágico"
    # Salto: 256 - 192 = 64
    # Subredes: 2^2 = 4
    # Hosts por subred: 2^6 - 2 = 62
    ```
  </TabItem>
  
  <TabItem label="Solución">
    ```bash title="Rangos de subredes"
    Subred 1: 192.168.1.0   - 192.168.1.63   (62 hosts)
    Subred 2: 192.168.1.64  - 192.168.1.127  (62 hosts)
    Subred 3: 192.168.1.128 - 192.168.1.191  (62 hosts)
    Subred 4: 192.168.1.192 - 192.168.1.255  (62 hosts)
    ```
  </TabItem>
</Tabs>

### Ejemplo 2: Red /27

<Tabs>
  <TabItem label="Problema">
    **Red:** 10.0.0.0/27
    **Máscara:** 255.255.255.224
    **Calcular:** Primeras 4 subredes
  </TabItem>
  
  <TabItem label="Solución">
    ```bash title="Primeras 4 subredes de 8 totales"
    # Salto: 256 - 224 = 32
    
    Subred 1: 10.0.0.0   - 10.0.0.31   (30 hosts)
    Subred 2: 10.0.0.32  - 10.0.0.63   (30 hosts)
    Subred 3: 10.0.0.64  - 10.0.0.95   (30 hosts)
    Subred 4: 10.0.0.96  - 10.0.0.127  (30 hosts)
    ```
  </TabItem>
</Tabs>

<Tabs>

## 2. Caso Empresarial Completo

<Badge text="Caso Real" variant="caution" size="small" />

### Escenario: Empresa Mediana

**Contexto:** Una empresa necesita dividir la red 192.168.100.0/24 para:
- 3 departamentos (Ventas, IT, Administración)
- Cada departamento necesita al menos 50 hosts
- Reservar espacio para crecimiento futuro

### Análisis Paso a Paso

<Steps>

1. **Análisis de Requerimientos**
   ```bash title="Evaluación inicial"
   # Red original: 192.168.100.0/24 (256 direcciones)
   # Subredes mínimas: 3
   # Hosts mínimos por subred: 50
   ```

2. **Búsqueda en Escalera de Potencias**
   ```bash title="Selección de subredes"
   # 2^1 = 2 subredes (insuficiente)
   # 2^2 = 4 subredes (suficiente + buffer) ✓
   ```

3. **Cálculo de Capacidad**
   ```bash title="Verificación de hosts"
   # Bits prestados: 2
   # Bits de host restantes: 32 - 26 = 6
   # Hosts por subred: 2^6 - 2 = 62 hosts ✓
   ```

4. **Implementación**
   ```bash title="Asignación final"
   # Nueva máscara: /24 + 2 = /26
   # Salto: 256 - 192 = 64
   
   Ventas:         192.168.100.0/26   (.1   - .62)
   IT:             192.168.100.64/26  (.65  - .126)
   Administración: 192.168.100.128/26 (.129 - .190)
   Reservada:      192.168.100.192/26 (.193 - .254)
   ```

5. **Verificación Matemática**
   ```bash title="Tests de validación"
   # Test de consistencia:
   4 subredes × 64 direcciones = 256 direcciones ✓
   
   # Test de factibilidad:
   nueva_máscara = 26 ≤ 30 ✓
   
   # Test de capacidad:
   62 hosts ≥ 50 hosts requeridos ✓
   ```

</Steps>

## 3. Casos por Tipo de Empresa

### Pequeña Empresa (< 50 empleados)

**Escenario:** Red 192.168.1.0/24 para 4 áreas

<Tabs>
  <TabItem label="Requerimientos">
    - **Administración:** 15 hosts
    - **Ventas:** 20 hosts  
    - **Técnico:** 10 hosts
    - **Invitados:** 8 hosts
  </TabItem>
  
  <TabItem label="Solución">
    ```bash title="División con /27 (30 hosts cada una)"
    # Usar /27 da 8 subredes de 30 hosts
    
    Administración: 192.168.1.0/27   (.1 - .30)
    Ventas:         192.168.1.32/27  (.33 - .62)
    Técnico:        192.168.1.64/27  (.65 - .94)
    Invitados:      192.168.1.96/27  (.97 - .126)
    Reservadas:     4 subredes adicionales para crecimiento
    ```
  </TabItem>
</Tabs>

### Empresa Mediana (50-200 empleados)

**Escenario:** Red 172.16.0.0/16 para múltiples oficinas

<Tabs>
  <TabItem label="Requerimientos">
    - **Oficina Central:** 500 hosts
    - **Oficina A:** 200 hosts
    - **Oficina B:** 150 hosts
    - **Servidores:** 50 hosts
    - **DMZ:** 20 hosts
  </TabItem>
  
  <TabItem label="Solución VLSM">
    ```bash title="Máscaras variables según necesidad"
    # Oficina Central: /22 (1022 hosts)
    172.16.0.0/22    (.1 - 172.16.3.254)
    
    # Oficina A: /24 (254 hosts)  
    172.16.4.0/24    (.1 - .254)
    
    # Oficina B: /24 (254 hosts)
    172.16.5.0/24    (.1 - .254)
    
    # Servidores: /26 (62 hosts)
    172.16.6.0/26    (.1 - .62)
    
    # DMZ: /27 (30 hosts)
    172.16.6.64/27   (.65 - .94)
    ```
  </TabItem>
</Tabs>

## 4. Verificaciones y Troubleshooting

### Tests de Validación

<Tabs>
  <TabItem label="Test de Conservación">
    ```bash title="Verificación matemática básica"
    # Para cualquier división:
    Subredes × Direcciones_por_subred = Total_original
    
    # Ejemplo /26 en /24:
    4 × 64 = 256 ✓
    ```
  </TabItem>
  
  <TabItem label="Test de No Solapamiento">
    ```bash title="Verificación de rangos"
    # Correcto:
    Subred 1: 192.168.1.0   - 192.168.1.63
    Subred 2: 192.168.1.64  - 192.168.1.127  ✓
    
    # Incorrecto (solapamiento):
    Subred 1: 192.168.1.0   - 192.168.1.63
    Subred 2: 192.168.1.32  - 192.168.1.95   ✗
    ```
  </TabItem>
  
  <TabItem label="Test de Factibilidad">
    ```bash title="Verificación de límites"
    # Reglas básicas:
    nueva_máscara ≤ 30 (para LANs con hosts)
    hosts_calculados ≥ hosts_requeridos
    subredes_calculadas ≥ subredes_requeridas
    ```
  </TabItem>
</Tabs>

### Errores Comunes

<Steps>

1. **Error de Cálculo de Salto**
   ```bash
   # Incorrecto: usar el valor de la máscara
   # Correcto: 256 - valor_máscara
   
   Para /26 (192): Salto = 256 - 192 = 64 ✓
   ```

2. **Olvidar Direcciones Reservadas**
   ```bash
   # Hosts reales = 2^bits - 2
   # No olvidar restar red y broadcast
   ```

3. **Exceder Límites de Máscara**
   ```bash
   # Para LANs: máximo /30 (2 hosts)
   # Para enlaces P2P: máximo /31 (RFC 3021)
   ```

</Steps>

## 5. Plantillas de Cálculo Rápido

### Fórmula de Verificación Rápida

```bash title="Cálculo mental directo"
# Para cualquier máscara /X:
Hosts = 2^(32-X) - 2

# Para encontrar máscara dado hosts:
X = 32 - log2(hosts + 2)

# Ejemplo: 100 hosts necesarios
X = 32 - log2(102) ≈ 32 - 6.67 ≈ 25.33 → /25
Verificación: 2^(32-25) - 2 = 126 hosts ✓
```
